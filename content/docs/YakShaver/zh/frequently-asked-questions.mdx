## 1. YakShaver 支持所有语言吗？（不仅仅是英语）

是的，YakShaver 支持世界上大多数语言。[查看支持的 OpenAI 列表](https://github.com/openai/whisper?tab=readme-ov-file#available-models-and-languages)。

查看由录音创建的问题示例：[中文](https://github.com/SSWConsulting/Northwind365/issues/162) 和 [法语](https://github.com/SSWConsulting/Northwind365/issues/168)。

---

## 2. 我需要认证人员在公司使用 YakShaver 吗？

SSW 建议您的组织中有了解 YakShaver 的人员。他们可以作为您的第一联络点，以防需要更新配置或出现任何异常。SSW 提供免费认证课程，成功完成后您的组织将获得价值 25 美元的 50 个免费积分。[点击这里申请](https://yakshaver.ai/certifications)。

---

## 3. 什么时候*不*应该使用 YakShaver？

YakShaver 非常适合提供上下文并准确展示用户体验。在您请求小的**文本更改**（改写、措辞改进或语气调整）且明确知道哪个项目和人员负责执行时，最好手动创建问题或 PBI。

在这些情况下，请遵循 SSW 规则["您是否请求从 X 到 Y 的小内容更改？"](https://www.ssw.com.au/rules/change-from-x-to-y)。这确保了清晰、意图明确，并促进与内容负责人更好的协作。

---

## 4. 我刚创建了一个工作项，这些按钮是做什么的？

其实很简单！

* **打开工作项：** 在您首选的浏览器中打开该问题
* **移动工作项：** 如果您在错误的项目中创建了工作项，可以将其移动到正确的项目
* **关闭工作项：** 如果您误创建了工作项，可以关闭该工作项

## 5. 为什么我的工作项中有这么多人被抄送？

创建工作项时，录音中提到的任何人都会自动加入对话。对于项目相关的事项，主要联系人和所有项目成员也会被标记，以确保团队的可见性和一致性。

---

## 6. 为什么我的 GitHub 问题是由 YakShaver 创建，而不是我的账户？

要在您的 GitHub 账户下创建和编辑问题，您需要将 GitHub 账户连接到 YakShaver。这确保问题正确归属于您，并赋予您完整的编辑权限。请参阅[连接您的 GitHub 账户](/docs/connect-your-github-account)获取设置说明。

---

## 7. 如何彻底从 GitHub 问题中删除机密信息？

如果您不小心在通过 YakShaver 录制的 GitHub 问题中包含了敏感内容（例如密码、私人数据），**仅编辑问题是不够的**。GitHub 会保留完整的修订历史。请参阅[如何彻底从 GitHub 问题中删除机密信息](https://www.ssw.com.au/rules/remove-confidential-information-from-github)。
