title: Lock status

## Overview

Every tenant in **Portal | Tenants** has a status that is either Unlocked (default) or Locked.

This setting is used by administrators to protect important customer tenants and control where certain system emails are sent. It does not affect how people use TimePRO on a day-to-day basis.

## Quick comparison

|                        | **Unlocked**                                                 | **Locked**                                                       |
| ---------------------- | ------------------------------------------------------------ | ---------------------------------------------------------------- |
| **Aim**                | Default status for new tenants and trials                    | Used for active production customers                             |
| **Manual decomission** | Can be decommissioned                                        | Cannot be decommissioned until unlocked                          |
| **Auto decomission**   | Can be automatically decommissioned after long inactivity    | Protected from automatic decommissioning                         |
| **Recurring invoice**  | Recurring invoice notifications are sent to the tenant owner | Recurring invoice notifications are sent to SSW account managers |
| **Typical use**        | Best for new sign-ups, trials, and temporary tenants         | Best for live customer tenants managed by SSW                    |

## When should a tenant be locked?

**Lock** a tenant when it becomes a live customer that SSW actively supports.

Leave it Standard (**unlocked**) if it is:

* A new sign-up
* A trial tenant
* Still being set up
* Likely to be decommissioned if abandoned

Note: All new tenants are created as Standard (Unlocked) by default.
